# Pilot Issue Categories

Use these categories in the Pilot Issue Log inside `/quality-home`.

## Blocker

Use when a pilot journey cannot continue.

Examples:
- Page cannot open.
- Save fails.
- Imported data cannot be used.
- Defect creation is blocked for allowed role.

## High

Use when the issue affects core pilot confidence but has a workaround.

Examples:
- Lookup mapping fills incomplete fields.
- Formula result is unclear.
- Workflow action is available to the wrong role.
- Command Center misses a real record.

## Medium

Use when the issue slows users down or creates operational confusion.

Examples:
- Field label is unclear.
- Empty state does not explain next action.
- Report wording needs adjustment.
- Export naming is confusing.

## Low

Use when the issue is minor and does not block pilot execution.

Examples:
- Cosmetic spacing issue.
- Long text wraps poorly.
- Button order could be improved.

## UX Improvement

Use for experience improvements discovered during pilot.

Examples:
- Need fewer clicks for inspectors.
- Need clearer management wording.
- Need a shortcut from one page to another.

## Data Issue

Use for real-data quality problems.

Examples:
- Missing part number.
- Duplicated defect names.
- Inconsistent model naming.
- Missing inspected quantity.

## Configuration Issue

Use for setup problems.

Examples:
- Wrong lookup source table.
- Missing formula reference.
- Incorrect role visibility.
- Inspection plan not published.

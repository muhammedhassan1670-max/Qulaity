دليل المستخدم ودليل الأدمن (QMS Platform)
================================

مقدمة
-----
هذا الدليل يشرح:
1) تعريف كل جزء/صفحة في النظام.
2) طريقة العمل كمستخدم (User).
3) طريقة العمل كأدمن (Admin).

ملاحظات عامة
------------
- بعض البيانات قد تكون تجريبية (Mock) حسب بيئة التشغيل.
- إعدادات الشارت (Charts) أصبحت ديناميكية ويمكن التحكم بها من الأدمن من صفحة واحدة.
- عند وجود زر Save في صفحات الأدمن: يجب الضغط عليه لحفظ التغييرات وتطبيقها.


A) صفحات المستخدم (User Sections)
===============================

1) Dashboard (لوحة التحكم الرئيسية)
---------------------------------
التعريف:
- ملخص مؤشرات الجودة والأداء ورسومات بيانية متعددة.

كمستخدم:
- افتح Dashboard لمتابعة المؤشرات والاتجاهات.
- استخدم الفلاتر (إن وُجدت) لتغيير نطاق العرض.
- راقب الرسومات لمعرفة التغيّر في المؤشرات خلال الوقت.


2) Executive Mode (لوحة تنفيذية)
-------------------------------
التعريف:
- عرض مركز للمؤشرات الأساسية (KPIs) مناسب للإدارة العليا.

كمستخدم:
- تابع الرسوم الاتجاهية ومؤشرات الأداء العامة.


3) IoT Real-Time Charts (رسومات لحظية)
--------------------------------------
التعريف:
- عرض قراءات حساسات لحظية/شبه لحظية مع إظهار الشذوذ (Anomalies) و/أو طبقة SPC حسب الإعداد.

كمستخدم:
- افتح الرسومات اللحظية لمراقبة القيمة مع الزمن.
- لاحظ النقاط الحمراء/المميزة (إن وُجدت) كدلائل شذوذ.


4) SPC System (نظام SPC)
------------------------
التعريف:
- نظام التحكم الإحصائي بالعملية (SPC) مع:
  - Control Limits: UCL / CL / LCL
  - Spec Limits: USL / LSL
  - Nelson Rules (قواعد لاكتشاف الانحرافات)
  - Characteristics (خصائص/قياسات) يتم اختيارها من بطاقات أعلى الصفحة.

كمستخدم:
- اختر Characteristic من البطاقات لعرض مخططها.
- راقب خطوط الحدود UCL/LCL وUSL/LSL.
- راقب Statistics (Mean, StdDev, Cp, Cpk) لفهم قابلية العملية.
- راقب Nelson Rules لمعرفة القواعد التي تم انتهاكها.


5) QualityOps Pro (المساعد الذكي الصناعي - AI Assistant)
--------------------------------------------------------
التعريف:
- مساعد ذكي (Enterprise AI) متخصص كهندس جودة متقدم، ومدير بيانات، ومدقق امتثال (ISO). يعتمد على محرك تحليل سياقي (Intent Detection Engine) ويفهم اللهجات المهنية والصناعية للشرق الأوسط.

كمستخدم:
- المحادثة المباشرة: اسأل عن أعطال الجودة، التحليلات (SPC, Pareto)، أو تقارير شاملة (CAPA, 8D, NCR).
- تحليل المستندات المتعددة (Document & Visual Intelligence):
  - الصور: ارفع صورة لأي عيب تصنيعي، وسيقوم المساعد بتحليل نوع العيب والأسباب المحتملة واقتراح خطط تصحيح.
  - ملفات البيانات (Excel/CSV): ارفع بيانات وسيقوم بتحليل الـ Cpk والـ SPC آلياً.
  - ملفات الامتثال (ISO): ارفع مستند وسيقوم بعمل مراجعة سريعة (Gap Analysis) للمستند.
- الاقتراحات الديناميكية (Dynamic Suggestions): استخدم الأزرار المقترحة السريعة للوصول لتقارير معقدة أسرع.


B) لوحة الأدمن (Admin Panel)
===========================

الدخول للأدمن
-------------
- من تبويب Admin/لوحة الأدمن ستجد أقسام الإدارة الأساسية (Users, Roles, Plants… إلخ).


1) Admin -> Charts (إعدادات الشارت الديناميكية)
----------------------------------------------
التعريف:
- صفحة مركزية للتحكم في إعدادات الشارت لكل أجزاء النظام بدون تعديل كود.

طريقة العمل كأدمن:
- عدّل الإعدادات (الألوان، الشذوذ التلقائي، حدود SPC، وغيرها).
- اضغط Save لتطبيق الإعدادات.


2) Admin -> Users (المستخدمين)
-----------------------------
التعريف:
- إدارة حسابات المستخدمين.


3) Admin -> Roles (الأدوار والصلاحيات)
--------------------------------------
التعريف:
- إدارة Roles وربطها بالصلاحيات.


4) Admin -> Plants / Multi-Plant (المصانع)
------------------------------------------
التعريف:
- إدارة مواقع/مصانع متعددة وإعداداتها الأساسية.


5) Admin -> Workflow (الأتمتة)
-----------------------------
التعريف:
- إعداد وإدارة Workflows لأتمتة إجراءات داخل النظام.


6) Admin -> Dynamic No-Code Form Builder (منشئ النماذج الذكي)
-------------------------------------------------------------
التعريف:
- أداة متقدمة جداً لبناء نماذج (Forms) جودة معقدة (مثل CAPA أو NCR) بدون استخدام كود برمجي، إما يدوياً (سحب وإفلات) أو عن طريق الذكاء المدمج لتحليل جداول البيانات.

كأدمن:
- البناء التلقائي عبر Excel/CSV الأتمتة: قم برفع ملف Excel أو CSV يحتوي على أعمدة للبيانات. سيقوم محرك البناء الديناميكي بالتعرف التلقائي على أنواع الحقول (نصوص، تواريخ، أرقام) وتحويلها لاستمارة في ثواني!
- البناء اليدوي (Drag & Drop): يمكنك سحب مكونات (نص قصير، نص طويل، قوائم منسدلة، تاريخ) وإسقاطها في صفحة العمل.
- التحرير والمعاينة الحية (Live Preview): يمكنك التعديل على الحقول، تغيير ترتيبها، وحذفها ورؤية النتيجة فوراً على اليمين.
- ربط النماذج: بعد الاعتماد والمراجعة، يتم حفظ النموذج ضمن الموديول المستهدف لاستخدامه في المنصة.


7) Admin -> Dropdowns (القوائم المنسدلة)
----------------------------------------
التعريف:
- إدارة القوائم المنسدلة (مثل المنتجات، أنواع العيوب… إلخ).


8) Admin -> Reports (التقارير)
------------------------------
التعريف:
- عرض وإدارة التقارير الإدارية.


9) Admin -> Database (قاعدة البيانات)
-------------------------------------
التعريف:
- شاشة معلومات/مراقبة عامة لحالة قاعدة البيانات.


ملاحظات دعم وتشخيص (Troubleshooting)
-----------------------------------
- إذا تغيّرت إعدادات الشارت ولم تظهر فورًا:
  - تأكد أنك ضغطت Save وحدّث الصفحة.
- بالنسبة للـ AI Assistant:
  - في حال رفع ملف Excel كبير، انتظر لحظات حتى ينتهي محرك التحليل من القراءة وتقديم التقييم.

11. AUTONOMOUS QUALITY ENGINE (QMS 4.0 INTEGRATION)
-----------------------------------------------
The platform is powered by an invisible background orchestrator (`QualityEngine`) that enforces the linkage between all core modules.

A. Issue Escalation (NCR -> CAPA -> 8D)
   - When an NCR hits critical priority or is escalated, the system automatically spins up a CAPA.
   - If the CAPA involves a customer complaint or is critical, it will automatically spin up an 8D.

B. Statistical Process Control (SPC -> NCR)
   - Real-time Chart Monitoring: Any point that drops below LCL or spikes above UCL will immediately trigger an Auto-NCR, halting the line.
   - AI Capability Monitoring (Cpk): If Cpk drops below 1.33, the system issues a warning and suggests Preventive Action. If Cpk < 1.0, it generates a critical NCR.

C. Risk Governance (FMEA -> Control Plan)
   - Reoccurring NCRs automatically scale up the "Occurrence" rating in related FMEA records.
   - If the Risk Priority Number (RPN) exceeds 100, the system automatically drafts a stricter Control Plan version to enhance inspection.

D. Change Control Synchronization
   - Approved Change Controls broadcast to all interconnected documents to ensure FMEA, CAPA, and Control Plans stay perfectly updated without manual typing.
